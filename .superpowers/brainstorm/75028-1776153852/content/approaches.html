Phase 7: 3D Depth Effects — Architecture Approaches

How to integrate 3D rendering into the existing fullscreen-quad + fragment shader pipeline

A

Unified Shader — Everything in Fragment

Raymarching for 7.1 shapes, parallax UV offset for 7.2, and a separate render path for 7.3 mesh distortion (swap fullscreen quad for subdivided grid when mesh mode is active).

Pros

  • 7.1 + 7.2 require zero engine changes — pure GLSL
  • Raymarching gives pixel-perfect smooth surfaces
  • Parallax is a simple UV shift per layer
  • Mesh distortion isolated to its own code path

Cons

  • Two geometry paths in engine (quad vs grid) adds complexity
  • Raymarching is expensive — ~50-100 steps per pixel per shape
  • Fragment shader grows by ~300 lines

Recommended — cleanest separation, leverages existing architecture for 7.1/7.2, only changes engine for 7.3

B

Always Mesh — Replace Quad with Grid

Replace the fullscreen quad with a 64x64 grid mesh for all rendering. 7.1 uses vertex displacement to approximate shapes, 7.2 offsets layer positions, 7.3 uses noise displacement. When no 3D is active, the mesh renders flat (identity).

Pros

  • Single geometry path — no conditional switching
  • All 3D effects share the same vertex pipeline
  • Mesh is always available for future features

Cons

  • 4K vertices on every frame even for 2D gradients (wasted GPU work)
  • Vertex-approximated sphere/torus looks faceted vs raymarched
  • Breaks the "fullscreen quad" invariant — risk of regressions in all existing modes
  • More invasive engine refactor
C

Multi-Pass Compositing — Separate 3D Pass

Render the gradient to an offscreen FBO first (existing pipeline), then sample that texture in a second pass that does raymarching/mesh distortion. Two-pass compositing.

Pros

  • Zero changes to existing gradient pipeline
  • 3D is a pure post-process — easy to toggle on/off
  • Gradient texture can be reused for multiple 3D views

Cons

  • Extra FBO + texture copy per frame (memory + bandwidth)
  • Complicates feedback loop (already uses FBO ping-pong)
  • Post-processing effects applied before or after 3D? Ordering gets messy
  • Parallax requires per-layer 3D — would need N FBOs for N layers
